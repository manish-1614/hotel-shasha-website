1. Overview &amp; Conventions
This document specifies all HTTP API endpoints exposed by the Shasha Jibhi web platform. In Phase 1, the platform exposes one primary API endpoint (/api/contact). This document also specifies the internal API contracts for server actions and future Phase 2 endpoints, so that frontend and backend work can proceed in parallel against a stable contract.

1.1 Base URL
Production:  https://shashajibhi.comPreview:     https://{branch-name}-shasha.vercel.appDevelopment: http://localhost:3000

1.2 Global Conventions
Convention
Specification
Protocol
HTTPS only. HTTP requests are redirected (301) to HTTPS.
Content-Type
All request and response bodies are application/json unless noted.
Authentication
Phase 1: None (public endpoints). Phase 2 admin routes: Bearer token (JWT) via Authorization header.
Rate Limiting
5 requests per IP per hour on all /api/* routes. Headers: X-RateLimit-Limit, X-RateLimit-Remaining, Retry-After.
Timestamps
All timestamps in ISO 8601 format: 2026-03-19T12:00:00.000Z (UTC).
Dates (user input)
Date-only fields (checkin, checkout) in YYYY-MM-DD format.
Currency
All monetary values in INR (Indian Rupees) as integers (paise not used; whole rupees only).
Error envelope
All errors return a consistent JSON envelope — see Section 3.
CORS
Not applicable — all API routes are same-origin. No CORS headers set.
Idempotency
POST /api/contact is effectively idempotent: duplicate submissions (same email + checkin within 24h) are silently accepted (200) but not re-processed.

1.3 Standard Error Envelope
All API error responses use this JSON structure:

{  &quot;error&quot;: {    &quot;code&quot;:    &quot;VALIDATION_ERROR&quot;,      // Machine-readable code (see Section 3)    &quot;message&quot;: &quot;Human-readable message&quot;,    &quot;details&quot;: { }                       // Optional: field-level errors (Zod)  }}
