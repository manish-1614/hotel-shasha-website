1. Executive Summary
Shasha: Paused Perfect is a boutique homestay in Jibhi, Himachal Pradesh вҖ” a serene Himalayan valley destination gaining rapid recognition among premium travellers seeking authentic mountain experiences. Operated by two brothers with deep roots in Himalayan adventure travel, Shasha offers an extraordinary blend of luxury accommodation, global cuisine, and human warmth.

This PRD v2.0 defines the comprehensive technical and product architecture for shashajibhi.com вҖ” a next-generation web platform engineered to serve as the property&apos;s primary digital identity, visual portfolio, lead-capture engine, and вҖ” in Phase 2 вҖ” a fully autonomous booking system. The platform is designed around five core pillars:

Immersive Visual Experience: A high-fidelity, performance-optimised gallery communicating the property&apos;s premium aesthetic through curated photography.
Brand Storytelling: An editorial-quality narrative journey converting visitors into high-intent enquiries through emotional resonance.
Intelligent Lead Capture: A 3-step progressive form that qualifies leads, collects stay preferences, and blocks spam and low-intent submissions.
Autonomous Booking Engine (Phase 2): Real-time availability, instant price quotes, and payment processing вҖ” eliminating manual host intervention.
Operational Observability (Phase 1): Analytics funnels, error monitoring, and structured logging operational from day one.

Core stack: Next.js 14 App Router В· TypeScript В· pnpm В· Tailwind CSS В· shadcn/ui В· Framer Motion В· Cloudinary CDN В· Resend В· NeonDB В· Vercel В· PostHog В· Sentry В· Axiom

5. Multi-Step Lead Capture Form вҖ” /contact
The 3-step progressive form remains the primary conversion path in Phase 1. From Phase 2, it co-exists with the booking engine вҖ” guests who prefer to speak to a human use the form; guests who want instant confirmation use /book.

5.1 Anti-Spam Architecture
Honeypot field: CSS-hidden &lt;input name=&apos;website&apos; tabindex=&apos;-1&apos;&gt; вҖ” bots fill it, humans don&apos;t.
Cloudflare Turnstile: Non-intrusive CAPTCHA on Step 3. Token verified server-side via /siteverify.
Rate limiting: Vercel Edge Middleware вҖ” 5 requests / IP / hour on /api/contact.
Duplicate detection: SHA-256 of (email + checkin) stored in Vercel KV (TTL 24h). Duplicates silently discarded.

7.1 Problem Statement
The Phase 1 platform collects enquiries via the contact form. The host then manually responds via WhatsApp to confirm availability and collect payment. This is viable below ~15 bookings/month but fails at scale: guests expect instant confirmation, and manual coordination creates errors, delays, and lost conversions. Phase 2 resolves this entirely.

рҹ“Ң  Design Goal:  Enable guests to check real-time room availability, receive an instant price quote, and complete payment вҖ” without host intervention. The host receives a confirmed booking notification, not a request to manually respond.

   рҹ”„  SECTION 8: OTA Synchronisation Strategy   

8.1 The Double-Booking Problem
Without OTA synchronisation, a guest can simultaneously book the Duplex Cottage on the Shasha website and on Airbnb for the same dates. The host must cancel one вҖ” incurring Airbnb penalties, negative reviews, and a full refund obligation. This is existentially damaging for a boutique property and must be solved before any OTA listing goes live.

вҡ   Risk:  A single double-booking incident costs 3вҖ“5Г— the booking value in refunds, penalties, and lost trust. Solve this before accepting any OTA listing.

   рҹ“қ  SECTION 9: CMS Strategy вҖ” Sanity   

9.1 Why Static Constants Are Not Sustainable
In PRD v1.0, all content lives in TypeScript constants files вҖ” rooms.ts, imageManifest.ts, testimonials.ts. Any content change requires: editing a .ts file вҶ’ Git commit вҶ’ Vercel build (3вҖ“4 min) вҶ’ production deploy.    рҹ“Ҡ  SECTION 10: Analytics Architecture вҖ” PostHog   

10.1 Why Vercel Analytics Alone Is Insufficient
Vercel Analytics provides Core Web Vitals (LCP, CLS, INP) and page view counts. It does not provide: which rooms guests view most, where they abandon the 3-step contact form, which traffic sources convert to enquiries, or session recordings to diagnose UX problems. PostHog fills every gap.

вҡЎ  Implement Now:  PostHog setup takes &lt; 2 hours. Installing it before launch means funnel data starts accumulating from day one. Retrofitting analytics 3 months later means 3 months of lost conversion data.

13. Key Component Specifications
13.1 &lt;GalleryGrid /&gt;
Props: images: ImageManifest[], initialFilter?: Category.
CSS columns masonry (column-count: 2 mobile, 3 tablet, 4 desktop) вҖ” no JS layout calculations.
IntersectionObserver adds 12 images when sentinel enters viewport. Image onLoad reveals with opacity transition.

13.2 &lt;MultiStepForm /&gt;
useReducer state machine: idle вҶ’ step1 вҶ’ step2 вҶ’ step3 вҶ’ submitting вҶ’ success | error.
React Hook Form + Zod resolver. Per-step schema validation. Framer Motion AnimatePresence for transitions.
aria-current=&apos;step&apos;, focus management on step transitions, role=&apos;alert&apos; on errors.

13.3 &lt;BookingCalendar /&gt; [NEW v2.0]
Renders a month-view calendar using react-day-picker showing available (green), unavailable (grey), and selected (forest) dates.
Availability data fetched from /api/availability on mount and on month navigation. Redis cache ensures sub-100ms responses.
On date selection, calls checkAvailability(); shows instant &apos;Available&apos; or &apos;Unavailable вҖ” next available: {date}&apos; feedback.

13.4 &lt;RoomCard /&gt;
Variants: &apos;preview&apos; (home page) | &apos;featured&apos; (stay page) | &apos;selector&apos; (form Step 2) | &apos;booking&apos; (availability calendar entry point) [NEW v2.0].

16. Security
All secrets in Vercel Environment Variables вҖ” never in codebase. .env.example committed (no values).
CSP headers via next.config.ts: whitelist Cloudinary, Resend, Turnstile, Google Fonts, Sentry, PostHog, Axiom domains.
Input sanitisation server-side before DB write or email composition.
HTTPS enforced by Vercel. HSTS header enabled. X-Frame-Options: DENY.
Razorpay webhook: HMAC-SHA256 signature verified before any DB write.
Sentry session replay: maskAllInputs: true. PII scrubbed in beforeSend().
Rate limiting: Upstash Redis + Vercel Edge Middleware on all /api/* routes.
