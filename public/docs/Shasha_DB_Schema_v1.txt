1. Overview &amp; Design Decisions
This document specifies the complete PostgreSQL database schema for the Shasha Jibhi platform. The database is introduced in Phase 2 using NeonDB (serverless PostgreSQL) with Drizzle ORM for type-safe queries. Phase 1 operates without a database — all enquiry data flows through email only.

Attribute
Value
Engine
PostgreSQL 16 (NeonDB serverless)
ORM
Drizzle ORM v0.30+ — schema-first, type-safe, no runtime overhead
Connection
Neon serverless driver (@neondatabase/serverless) with connection pooling
Migrations
Drizzle Kit — drizzle-kit generate:pg + drizzle-kit push:pg
Schema file
src/db/schema.ts (single file, all tables)
Naming convention
snake_case for tables and columns · plural table names · id as primary key
Primary keys
UUID v7 (time-ordered) via gen_random_uuid() — prefixed for readability (enq_, room_, etc.)
Timestamps
created_at + updated_at on all tables · auto-managed via Drizzle timestamps()
Soft deletes
deleted_at timestamp where applicable — no hard deletes on user-facing records
Data residency
NeonDB us-east-1 region (Phase 2 may migrate to Singapore for latency)

⚠  The database is NOT used in Phase 1. All schema and DDL in this document is forward-looking for Phase 2 implementation. The schema is designed to be extensible — Phase 3 tables are documented but not deployed until needed.

3. Table Schemas — Full DDL
3.1 enquiries
The primary data table. Every successfully processed contact form submission creates one record. This is the source of truth for all guest leads.

-- enquiries table DDLCREATE TABLE enquiries (  id              TEXT        PRIMARY KEY DEFAULT &apos;enq_&apos; || replace(gen_random_uuid()::text, &apos;-&apos;, &apos;&apos;),  -- Guest personal details  name            VARCHAR(80)  NOT NULL,  email           VARCHAR(255) NOT NULL,  phone           VARCHAR(15)  NOT NULL,  city            VARCHAR(60),  -- Stay preferences  room_type       room_type_enum NOT NULL,  room_id         TEXT         REFERENCES rooms(id),  checkin         DATE         NOT NULL,  checkout        DATE         NOT NULL,  nights          INTEGER      GENERATED ALWAYS AS (checkout - checkin) STORED,  guests          SMALLINT     NOT NULL CHECK (guests BETWEEN 1 AND 8),  meal_plan       meal_plan_enum NOT NULL,  requests        TEXT         CHECK (char_length(requests) &lt;= 400),  source          source_enum,  -- Computed financials  estimated_revenue INTEGER,   -- in INR, calculated at insert time  -- CRM workflow  status          enquiry_status_enum NOT NULL DEFAULT &apos;new&apos;,  host_notes      TEXT,        -- internal notes from host  -- Privacy &amp; audit  ip_hash         VARCHAR(64), -- SHA-256 of IP — for dedup analytics only  -- Timestamps  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),  deleted_at      TIMESTAMPTZ             -- soft delete);-- Enum definitionsCREATE TYPE room_type_enum AS ENUM (  &apos;duplex-triple&apos;, &apos;duplex-quad&apos;, &apos;private&apos;, &apos;dorm&apos;);CREATE TYPE meal_plan_enum AS ENUM (  &apos;room-only&apos;, &apos;cp-plan&apos;, &apos;full-board&apos;);CREATE TYPE source_enum AS ENUM (  &apos;instagram&apos;, &apos;google&apos;, &apos;friend&apos;, &apos;blog&apos;, &apos;ota&apos;, &apos;other&apos;);CREATE TYPE enquiry_status_enum AS ENUM (  &apos;new&apos;, &apos;contacted&apos;, &apos;confirmed&apos;, &apos;cancelled&apos;, &apos;completed&apos;);

3.2 rooms
Static room catalog. In Phase 1, room data lives in src/data/rooms.ts. Phase 2 migrates this to the database for admin-managed updates without code deployments.

CREATE TABLE rooms (  id          TEXT        PRIMARY KEY DEFAULT &apos;room_&apos; || replace(gen_random_uuid()::text, &apos;-&apos;, &apos;&apos;),  slug        VARCHAR(60)  NOT NULL UNIQUE, -- URL slug: duplex-cottage, private-room, dorm  name        VARCHAR(100) NOT NULL,  type        room_type_enum NOT NULL,  capacity    SMALLINT     NOT NULL,        -- max guests  description TEXT,  features    TEXT[]       NOT NULL DEFAULT &apos;{}&apos;, -- e.g. {&apos;Skylight roof&apos;,&apos;Valley view&apos;}  price_base  INTEGER      NOT NULL,        -- base price room-only in INR  is_active   BOOLEAN      NOT NULL DEFAULT true,  sort_order  SMALLINT     NOT NULL DEFAULT 0,  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now());

3.3 rate_configs
Seasonal pricing configurations. Supports multiple active rate tiers (standard, peak, off-season) with date ranges. The application selects the most specific active rate for a given checkin date.

CREATE TYPE rate_season_enum AS ENUM (&apos;standard&apos;, &apos;peak&apos;, &apos;offseason&apos;);CREATE TABLE rate_configs (  id                  TEXT        PRIMARY KEY DEFAULT gen_random_uuid()::text,  room_type           room_type_enum NOT NULL,  season              rate_season_enum NOT NULL DEFAULT &apos;standard&apos;,  price_room_only     INTEGER      NOT NULL,        -- INR per night  price_cp_per_person INTEGER      NOT NULL,        -- INR per person per night (CP plan)  valid_from          DATE         NOT NULL,  valid_to            DATE         NOT NULL,  is_active           BOOLEAN      NOT NULL DEFAULT true,  created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),  updated_at          TIMESTAMPTZ NOT NULL DEFAULT now(),  CONSTRAINT valid_date_range CHECK (valid_to &gt; valid_from));

3.4 gallery_images
Managed image catalog. In Phase 1, this is a static TypeScript file (imageManifest.ts). Phase 2 persists it here for admin-managed ordering, alt text, and featured image selection.

CREATE TYPE image_category_enum AS ENUM (  &apos;food&apos;, &apos;rooms-interior&apos;, &apos;rooms-lifestyle&apos;, &apos;exterior-drone&apos;, &apos;exterior-ground&apos;);CREATE TABLE gallery_images (  id             TEXT        PRIMARY KEY DEFAULT gen_random_uuid()::text,  cloudinary_id  VARCHAR(255) NOT NULL UNIQUE, -- e.g. shasha/food/2Y4A8137  category       image_category_enum NOT NULL,  alt_text       VARCHAR(255) NOT NULL,  featured       BOOLEAN      NOT NULL DEFAULT false,  sort_order     SMALLINT     NOT NULL DEFAULT 0,  width          SMALLINT,    -- original pixel width (for aspect ratio)  height         SMALLINT,    -- original pixel height  is_active      BOOLEAN      NOT NULL DEFAULT true,  created_at     TIMESTAMPTZ NOT NULL DEFAULT now());

3.5 testimonials

CREATE TYPE testimonial_source_enum AS ENUM (&apos;direct&apos;, &apos;google&apos;, &apos;booking&apos;, &apos;instagram&apos;);CREATE TABLE testimonials (  id            TEXT        PRIMARY KEY DEFAULT gen_random_uuid()::text,  guest_name    VARCHAR(80)  NOT NULL,  stay_type     room_type_enum,              -- which room they stayed in  quote         TEXT         NOT NULL CHECK (char_length(quote) &lt;= 500),  rating        SMALLINT     NOT NULL CHECK (rating BETWEEN 1 AND 5),  source        testimonial_source_enum NOT NULL DEFAULT &apos;direct&apos;,  is_published  BOOLEAN      NOT NULL DEFAULT false, -- must be explicitly published  created_at    TIMESTAMPTZ NOT NULL DEFAULT now());

3.6 blocked_dates  (Phase 3)

CREATE TABLE blocked_dates (  id          TEXT        PRIMARY KEY DEFAULT gen_random_uuid()::text,  room_id     TEXT         NOT NULL REFERENCES rooms(id) ON DELETE CASCADE,  date_from   DATE         NOT NULL,  date_to     DATE         NOT NULL,  reason      VARCHAR(255),                -- &apos;Maintenance&apos;, &apos;Personal stay&apos;, etc.  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),  CONSTRAINT valid_block_range CHECK (date_to &gt;= date_from));

3.7 admin_users  (Phase 2)

CREATE TYPE admin_role_enum AS ENUM (&apos;super_admin&apos;, &apos;host&apos;);CREATE TABLE admin_users (  id              TEXT        PRIMARY KEY DEFAULT gen_random_uuid()::text,  email           VARCHAR(255) NOT NULL UNIQUE,  password_hash   VARCHAR(255) NOT NULL, -- bcrypt, cost factor 12  role            admin_role_enum NOT NULL DEFAULT &apos;host&apos;,  last_login_at   TIMESTAMPTZ,  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now());-- Seed: initial admin users (run once during Phase 2 setup)-- Passwords set via admin setup script — never stored in codebase
4. Indexes
All indexes are documented with their purpose and the queries they optimise. Over-indexing on a small dataset is avoided — only indexes that serve real query patterns are created.

-- enquiries table indexesCREATE INDEX idx_enquiries_email  ON enquiries(email)  WHERE deleted_at IS NULL;-- Purpose: Deduplication lookup by email + checkinCREATE INDEX idx_enquiries_checkin  ON enquiries(checkin)  WHERE deleted_at IS NULL;-- Purpose: Filter enquiries by date range in admin dashboardCREATE INDEX idx_enquiries_status  ON enquiries(status)  WHERE deleted_at IS NULL;-- Purpose: Filter by status (new/contacted/confirmed)CREATE INDEX idx_enquiries_created_at  ON enquiries(created_at DESC)  WHERE deleted_at IS NULL;-- Purpose: Default sort in admin dashboard (newest first)CREATE UNIQUE INDEX idx_enquiries_dedup  ON enquiries(email, checkin)  WHERE deleted_at IS NULL;-- Purpose: DB-level deduplication constraint (belt + suspenders with Redis dedup)-- rooms table indexesCREATE UNIQUE INDEX idx_rooms_slug ON rooms(slug) WHERE is_active = true;-- Purpose: URL slug lookup: /stay/duplex-cottage → rooms WHERE slug=&apos;duplex-cottage&apos;-- rate_configs table indexesCREATE INDEX idx_rate_configs_room_type_dates  ON rate_configs(room_type, valid_from, valid_to)  WHERE is_active = true;-- Purpose: Find applicable rate for a given room + checkin date-- gallery_images table indexesCREATE INDEX idx_gallery_images_category  ON gallery_images(category, sort_order)  WHERE is_active = true;-- Purpose: Gallery filter by category with sort orderCREATE INDEX idx_gallery_images_featured  ON gallery_images(featured)  WHERE featured = true AND is_active = true;-- Purpose: Fetch featured images for homepage / room pages-- testimonials table indexesCREATE INDEX idx_testimonials_published  ON testimonials(is_published, created_at DESC)  WHERE is_published = true;-- Purpose: Fetch published testimonials for homepage carousel
